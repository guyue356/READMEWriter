# README 各章节详细写作指南

## 章节顺序原则

README 章节应遵循「先上手、后深入」的阅读顺序：

```text
项目简介 → 核心能力 → 效果展示 → 应用场景
→ 安装部署 → 快速开始 → 使用说明     ← 用户能跑起来
→ 系统架构 → 核心工作流程 → 技术栈    ← 感兴趣再深入
→ 配置说明 → 项目亮点 → Roadmap → 贡献指南 → FAQ → License
```

核心原则：**安装和使用排在架构前面**。大多数用户只想快速跑起来，不想先看 Mermaid 图。

---

## 1. 项目名称 + 一句话描述

**格式**：
```markdown
# 项目名称

> 一句话描述项目核心价值。
```

**要求**：
- 一句话不超过 30 字
- 突出"做什么"和"为什么"
- 示例：`基于 AI 的 GitHub 项目自动视频生成系统，通过代码理解与故事生成能力，将开源仓库自动转换为高质量产品介绍视频。`

---

## 2. 项目简介

**回答的核心问题**：这个项目为什么存在？

**必须包含**：
- 项目背景（行业/技术背景）
- 要解决的具体问题
- 为什么这个问题重要（痛点描述）
- 目标用户是谁
- 项目带来的核心价值

**写作要点**：
- 从问题出发，不要从技术出发
- 用具体场景说明痛点，不要用抽象描述
- 篇幅控制在 150-300 字

---

## 3. 核心能力

**格式**：每个功能用三级标题，包含功能描述 + 解决的问题 + 带来的价值。

```markdown
### 功能名称

**功能**：做什么
**解决的问题**：为什么需要
**价值**：带来什么好处
```

**要求**：
- 列出 3-6 个核心功能
- 按重要性排序
- 每个功能 2-3 句话

## 6. 安装部署

**位置**：必须排在「应用场景」之后、「系统架构」之前。用户想快速上手，不应被技术细节挡住。

**必须包含**：
- 环境要求（语言版本、依赖工具）
- 依赖安装命令
- 环境变量配置（列出所有必需变量）
- 数据库初始化（如需要）
- Docker 部署方式
- 验证安装是否成功的方法

**安装方式排序**（按难度递增）：
1. **让 AI 自己安装**（最简单）— 提供一段提示词，用户粘贴给 AI 工具即可自动完成安装
2. **脚本自动安装** — 提供安装脚本命令
3. **手动安装** — 提供手动复制/配置步骤

**AI 自安装提示词示例**（适用于工具/插件类项目）：
```
请帮我安装 XXX 技能：
1. 克隆仓库：git clone <repo-url>
2. 将技能目录复制到你的技能目录中
3. 确认安装成功后告诉我
```

## 7. 快速开始

**目标**：让用户 5 分钟内跑起来。
**原则**：不重复安装章节的内容，只给最短路径。如果安装章节已详细说明，快速开始只需 2-3 行指向安装章节 + 核心触发命令。

提供最短路径示例，通常是：
```bash
git clone <repo>
cd <repo>
cp .env.example .env  # 编辑配置
docker compose up -d  # 或 npm install && npm start
```

## 9. 系统架构

**必须使用 Mermaid 图**。

**图的类型选择**：
| 场景 | 推荐图表 |
|------|---------|
| 整体架构 | `flowchart TD` 或 `graph TD` |
| 数据流 | `flowchart LR` |
| 服务拓扑 | `flowchart TD` + 子图 |
| 时序交互 | `sequenceDiagram` |
| 状态转换 | `stateDiagram-v2` |
| 类关系 | `classDiagram` |

**要求**：
- 每个模块用一句话说明职责
- 标注数据流向
- 标注模块间关系
